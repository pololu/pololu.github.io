5 ## Summary
6 
7 This is a Python 3 library that helps interface with
8 [Motoron Motor Controllers][motoron] from a Raspberry Pi.
9 It should also work on other computers as long as they have Python 3, I&sup2;C
10 and the [smbus2] library.
11 This library does **not** support Python 2.
12 
13 ## Getting started
14 
15 To use this library, you will also need to enable I&sup2;C, figure out
16 which I2C bus to use, set up the I&sup2;C device permissions properly
17 (so you do not have to use `sudo`), and connect the Motoron to your
18 Raspberry Pi's I&sup2;C bus. If you are not sure how to do those things,
19 see the "Getting started" sections of the [Motoron user's guide][guide].
20 
21 This library depends on Python 3 and the easiest way to download the library
22 is to use git. To install these programs on Raspberry Pi OS, run:
23 
24  sudo apt-get install git python3-dev python3-pip
25 
26 This library depends on [smbus2], which you can install by running:
27 
28  sudo pip3 install smbus2
29 
30 Finally, to download and install the library, run:
31 
32  git clone https://github.com/pololu/motoron-rpi.git
33  cd motoron-rpi
34  sudo python3 setup.py install

36 ## Examples
37 
38 Several example programs come with the library. They are single-file
39 Python programs that have names ending with `_example.py`.
40 
41 You can run an example program by typing the path to the example. For example,
42 if you are in the `motoron-rpi` directory, type `./simple_example.py` to run
43 that example.

45 ## Troubleshooting
46 
47  FileNotFoundError: [Errno 2] No such file or directory: '/dev/i2c-1'
48 
49 The error message above indicates that I&sup2;C bus 1 was not found.
50 For most users, the solution is to run `sudo raspi-config nonint do_i2c 0`
51 to enable I&sup2;C bus 1 and then reboot.
52 You can see which I&sup2;C busses are enabled by running `ls /dev/i2c*`.
53 If you have connected your Motoron to a different I2C bus, you should specify
54 the bus number when creating the MotoronI2C object. For example:
55 
56  mc = motoron.MotoronI2C(bus=123)

69 ## Command timeout
70 
71 By default, the Motoron will turn off its motors if it has not received a valid
72 command in the last 1.5 seconds. You can change the amount of time it
73 takes for the Motoron to time out using
74 motoron.MotoronI2C.set_command_timeout_milliseconds or you can disable the
75 feature using motoron.MotoronI2C.disable_command_timeout.
